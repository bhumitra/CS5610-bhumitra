As soon as the window size becomes less than 960px, this input bar becomes vertically aligned.



Here I made use of the media queries in CSS. The following peice of code produces this responsive behaviour according to the change in screen size. Different behaviour is defined for different types of screen size. (max-width/min-width). So as width decreases, the background color changes, and the image size also decreases. When width becomes too small, image is not shown. The field group and field label come into picture only when the page width is more than 960px by horizontally aligning the label and input field. Hence the responsive behaviour of the input field is acheived.

       
       @media screen and (min-width: 960px) {
          body {
                background-color: ThreeDLightShadow;
            }
            #image {
                width: 500px;
            }
            .field-group {
                clear: both;
            }
            .field-label {
                float: left;
            }
           
        }
        @media screen and (max-width: 960px) {
            body {
                background-color: lightblue;
            }
            #image {
                width: 420px;
            }
        }
        @media screen and (max-width: 720px) {
            #image {
                width: 380px;
            }
            body {
                background-color: lightgreen;
            }
        }
        @media screen and (max-width: 600px) {
            #image {
                width: 340px;
            }
            body {
                background-color: lightyellow;
            }
        }
        @media screen and (max-width: 450px) {
            #image {
                width: 300px;
            }
            body {
                background-color: lightsalmon;
            }
        }
        @media screen and (max-width: 350px) {
            #image {
                display: none;
            }
        }
